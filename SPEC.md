# 猫咪伴侣（CatCompanion）V1 规格说明

## 1. 产品定位
- 形态：macOS 原生桌面 AI 助理 + 电子宠物。
- 目标：在同一产品里同时提供
  - 健康提醒（喝水 / 站立走动 / 闭眼休息）
  - AI 对话与任务执行（文件操作 / 终端命令 / 浏览器自动化）
- 平台：macOS 14+，Apple Silicon。

## 2. 版本策略（先 Full，再 MAS）
### Full 版（优先实现）
- 站外分发（非 Mac App Store）。
- 能力全开：提醒 + AI 对话 + 本机执行工具。
- 架构：`主 App + 同 bundle 内置 Helper`（方案 2）。

### MAS 版（后续阉割版）
- 仅保留可通过审核的能力子集。
- 默认不开放高风险自动化能力（终端/跨 App 自动化等）。

## 3. 目标与非目标
### 目标
- 提供可用、可持续迭代的 1.0 基线。
- 云端模型优先，本地模型（Ollama）兜底。
- 第三方 skills 默认禁用，仅白名单启用。

### 非目标（当前阶段）
- 不在第一阶段做多端同步。
- 不在第一阶段做复杂计费/IAP。
- 不在第一阶段做 MAS 审核适配收敛。

## 4. 架构与能力边界
- 主 App：UI、提醒引擎、权限引导、策略配置。
- 内置 Helper：AI 推理编排、工具执行、隔离运行、审计输出。
- 通信：本机进程间通信（第一阶段采用 stdio 探针，后续可切 XPC）。

## 5. 模型策略
- 默认策略：`cloudPreferred`
- 主模型：闭源云模型（由用户配置 API 或订阅）
- 兜底模型：本地 Ollama 模型
- 失败回退：云失败时自动切到本地；本地失败时返回可解释错误。

## 6. 安全策略
- 高风险动作必须显式确认。
- 第三方 skills 默认关闭；白名单由产品审核后启用。
- 审计日志默认本地保存，禁止硬编码密钥。

## 7. 第一阶段交付范围
- 保留现有提醒功能与多语言（简中/繁中/英文/日语）。
- 新增 AI 助理配置模型（路由策略、模型、执行权限、skills 策略）。
- 新增 Helper 运行时管理（启动、健康探针、状态展示）。
- 新增最小 Agent 可执行程序，并接入 OpenClaw Gateway：
  - `config`：连接探测（health）
  - `ask`：通过 `chat.send` 获取回复
- 新增连续对话窗口（多轮文本会话，复用 Gateway `sessionKey`）。
- 新增本地语音能力（CosyVoice 脚本桥接、设置项、离线依赖探测）。
- 新增首次运行诊断向导（Helper/Gateway/语音环境/麦克风权限检测）。

## 8. 里程碑
- M1：方案 2 架构基线（本次）
- M2：接入 OpenClaw Gateway 协议与连续对话链路
- M2.5：离线语音（whisper.cpp + CosyVoice）接入
- M3：Full 版封装分发（DMG/安装器）
- M4：MAS 阉割版能力裁剪与上架准备
